# Тест-кейсы для метода POST {{baseUrl}}/api/1/item (Сохранить объявление)

## Предусловия

- **Эндпоинт:** `{{baseUrl}}/api/1/item`
- **Метод:** `POST`
- **Заголовки:** `Content-Type: application/json`

**Схема 1:** Схема тела POST запроса на `{{baseUrl}}/api/1/item`.

**Схема 2:** Схема успешного ответа 200 OK для POST `{{baseUrl}}/api/1/item`.

**Схема 3:** Схема ответа об ошибке 400 Bad Request.

**Схема 1:**
```json
{
  "sellerID": "<integer>",
  "name": "<string>",
  "price": "<integer>",
  "statistics": {
    "likes": "<integer>",
    "viewCount": "<integer>",
    "contacts": "<integer>"
  }
}
```

**Схема 2:**
```json
{
  "id": "<string>",
  "sellerId": "<integer>",
  "name": "<string>",
  "price": "<integer>",
  "statistics": {
    "likes": "<integer>",
    "viewCount": "<integer>",
    "contacts": "<integer>"
  },
  "createdAt": "<string>"
}
```

**Схема 3:**
```json
{
  "result": {
    "messages": {
      "culpa_b92": "<string>",
      "enim_24f": "<string>",
      "mollit_aa": "<string>"
    },
    "message": "<string>"
  },
  "status": "<string>"
}
```

## 1. Позитивные тесты

| №      | Описание | Тело запроса (JSON) | Ожидаемый результат |
|:-------|:---------|:--------------------|:--------------------|
| **TC-01** | Сформировать HTTP-запрос в соответствии со **схемой 1** с валидными данными для каждого поля | `{ "sellerID": 111303, "name": "Test Item", "price": 1000, "statistics": { "likes": 5, "viewCount": 100, "contacts": 10 } }` | Сервер возвращает HTTP статус-код `200 OK`. Структура ответа соответствует **схеме 2**. Значения полей тела ответа соответствуют значениям полей из тела запроса. Тело ответа содержит поля `id` формата UUID и `createdAt` с датой создания объявления. |
| **TC-02** | Повторно создать объявление с теми же данными и убедиться что ID уникален | `{ "sellerID": 111303, "name": "Test Item", "price": 1000, "statistics": { "likes": 5, "viewCount": 100, "contacts": 10 } }` (два одинаковых запроса подряд) | Сервер возвращает HTTP статус-код `200 OK` для каждого запроса. Поля `id` в обоих ответах различаются. |
| **TC-03** | Сформировать HTTP-запрос в соответствии со **схемой 1** с нулевыми значениями допустимых полей (`price`, `likes`, `viewCount`, `contacts`) | `{ "sellerID": 111303, "name": "Test Item", "price": 0, "statistics": { "likes": 0, "viewCount": 0, "contacts": 0 } }` | Сервер возвращает HTTP статус-код `200 OK`. Структура ответа соответствует **схеме 2** с нулевыми значениями в соответствующих полях. Значения полей тела ответа соответствуют значениям полей из тела запроса. |
| **TC-05** | Создать несколько объявлений с одним `sellerID` | `{ "sellerID": 111303, "name": "First Item", "price": 1000, ... }`, затем `{ "sellerID": 111303, "name": "Second Item", "price": 500, ... }` | Сервер возвращает HTTP статус-код `200 OK` для каждого запроса. Объявления получают разные `id`. Поле `sellerId` в обоих ответах совпадает с переданным `sellerID`. |
