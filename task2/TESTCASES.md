# Тест-кейсы для метода POST {{baseUrl}}/api/1/item (Сохранить объявление)

## Предусловия

- **Эндпоинт:** `{{baseUrl}}/api/1/item`
- **Метод:** `POST`
- **Заголовки:** `Content-Type: application/json`

**Схема 1:** Схема тела POST запроса на `{{baseUrl}}/api/1/item`.

**Схема 2:** Схема успешного ответа 200 OK для POST `{{baseUrl}}/api/1/item`.

**Схема 3:** Схема ответа об ошибке 400 Bad Request.

**Схема 1:**
```
{
  "sellerID": "<integer>",
  "name": "<string>",
  "price": "<integer>",
  "statistics": {
    "likes": "<integer>",
    "viewCount": "<integer>",
    "contacts": "<integer>"
  }
}
```

**Схема 2:**
```
{
  "id": "<string>",
  "sellerId": "<integer>",
  "name": "<string>",
  "price": "<integer>",
  "statistics": {
    "likes": "<integer>",
    "viewCount": "<integer>",
    "contacts": "<integer>"
  },
  "createdAt": "<string>"
}
```

**Схема 3:**
```
{
  "result": {
    "messages": {
      "culpa_b92": "<string>",
      "enim_24f": "<string>",
      "mollit_aa": "<string>"
    },
    "message": "<string>"
  },
  "status": "<string>"
}
```

| №      | Описание | Тело запроса (JSON) | Ожидаемый результат |

| **TC-01** | Сформировать HTTP-запрос в соответствии со **схемой 1** с валидными данными для каждого поля | `{ "sellerID": 111303, "name": "Test Item", "price": 1000, "statistics": { "likes": 5, "viewCount": 100, "contacts": 10 } }` | Сервер возвращает HTTP статус-код `200 OK`. Структура ответа соответствует **схеме 2**. Значения полей тела ответа соответствуют значениям полей из тела запроса.
