# Баг-репорты

## Критичность: High

### 1: API: `POST /api/1/item` принимает отрицательные значения в `statistics`

| Поле                         | Значение |
|:-----------------------------| :--- |
| **Приоритет**                | **High** |
| **Шаги для воспроизведения** | 1. Отправить `POST` запрос на `/api/1/item` с отрицательными значениями в `statistics` (`likes`, `viewCount`, `contacts`). |
| **Фактический результат**    | Сервер возвращает `200 OK` и создает объявление с отрицательной статистикой. |
| **Ожидаемый результат**      | Сервер должен вернуть `400 Bad Request`, так как значения статистики не могут быть отрицательными. |

---

### 2: API: `POST /api/1/item` принимает отрицательный `price`

| Поле                         | Значение |
|:-----------------------------| :--- |
| **Приоритет**                | **High** |
| **Шаги для воспроизведения** | 1. Отправить `POST` запрос на `/api/1/item` с `price < 0`. |
| **Фактический результат**    | Сервер возвращает `200 OK` и создает объявление с отрицательной ценой. |
| **Ожидаемый результат**      | Сервер должен вернуть `400 Bad Request`, так как цена не может быть отрицательной. |

---

## Критичность: Medium

### 3: API: `POST /api/1/item` возвращает некорректный формат ответа

| Поле                         | Значение |
|:-----------------------------| :--- |
| **Приоритет**                | **Medium** |
| **Шаги для воспроизведения** | 1. Отправить валидный `POST` запрос на `/api/1/item`.<br>2. Проанализировать тело ответа. |
| **Фактический результат**    | Сервер возвращает `{ "status": "Сохранили объявление - <id>" }`. |
| **Ожидаемый результат**      | Сервер должен вернуть полноценный JSON-объект объявления с полями `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`. |

---

### 4: API: `POST /api/1/item` отклоняет валидные нулевые значения в `statistics`

| Поле                         | Значение |
|:-----------------------------| :--- |
| **Приоритет**                | **Medium** |
| **Шаги для воспроизведения** | 1. Отправить `POST` запрос с `statistics`, где все значения равны `0`. |
| **Фактический результат**    | Сервер возвращает `400 Bad Request`. |
| **Ожидаемый результат**      | Сервер должен вернуть `200 OK`, так как нулевые значения валидны для новой статистики. |

---

### 5: API: `POST /api/1/item` принимает недокументированное поле `sellerId`

| Поле                         | Значение |
|:-----------------------------| :--- |
| **Приоритет**                | **Medium** |
| **Шаги для воспроизведения** | 1. Отправить `POST` запрос, используя `sellerId` вместо `sellerID`. |
| **Фактический результат**    | Сервер возвращает `200 OK` и создает объявление. |
| **Ожидаемый результат**      | Сервер должен вернуть `400 Bad Request` и принимать только документированные поля. |

---
